# Mesh Filtering OpenMVG/OpenMVS Post-Processing

A post-processing script that cleans and optimizes 3D meshes generated by the **OpenMVG** (Structure from Motion) and **OpenMVS** (Multi-View Stereo) pipelines. It uses camera pose information to automatically determine a projection plane, filter out spurious geometry, and optionally decimate the mesh.

---

## Overview

<div align="center">

![Overview of mesh filtering process](overview/cut_process.gif)

</div>

The video above shows the full pipeline in action:

1. **First mesh obtained** - the raw dense mesh produced by OpenMVS, which typically contains a large amount of spurious geometry outside the area of interest (floor, walls, background elements reconstructed from partial views).
2. **Estimated camera points** - the 3D positions of all cameras read from `sfm_data.json`. These positions define the capture volume: the region of the scene that was actually observed.
3. **Projected camera points** - the camera centers projected orthogonally onto the plane perpendicular to the dominant camera axis, making all points coplanar.
4. **Convex hull from projected camera points** - the convex hull of the projected points, defining a 3D polygon that tightly encloses the observed area and will be used to obtain the cut plane.
5. **Cut plane** - the plane derived from the convex hull, positioned and oriented to separate the geometry of interest from the surrounding noise.
6. **Cut mesh** - the result of filtering the original mesh against the cut plane, retaining only the faces whose vertices fall inside the bounding polygon.
7. **Optimized mesh** - the final mesh after removing small disconnected components, keeping the largest connected piece, and optionally applying decimation to reduce face count.

## Input files

If you would like to test the code, all required input files can be found in the `material/` folder:

| File | Origin | Description |
|---|---|---|
| `scene_dense_mesh.obj` | OpenMVS | Dense 3D mesh to be filtered and optimized |
| `sfm_data.json` | OpenMVG | Scene file in JSON format containing camera poses (rotation matrices and centers) for each reconstructed view |

### Generating `sfm_data.json`

The file is obtained by converting the binary OpenMVG scene file using the official format converter from OpenMVG:

```bash
openMVG_main_convertSfM_DataFormat \
  -i /path/to/sfm_data.bin \
  -o /path/to/sfm_data.json
```

The script reads the `extrinsics` section of the JSON, using the first row of each camera's rotation matrix to compute the dominant camera axis.

## Usage

```bash
python mesh_filtering.py -- <sfm_data_path> <mesh_path> <output_folder> [num_polygons]
```

### Arguments

| Position | Name | Description |
|---|---|---|
| 1 | `sfm_data_path` | Path to `sfm_data.json` |
| 2 | `mesh_path` | Path to `scene_dense_mesh.obj` (read-only, the original file is **not** modified) |
| 3 | `output_folder` | Directory where intermediate and final files are saved |
| 4 *(optional)* | `num_polygons` | Target face count for decimation. `0` or omitted = no decimation |

### Example

```bash
python mesh_filtering.py -- material/sfm_data.json material/scene_dense_mesh.obj output/
```

With decimation to 50 000 faces:

```bash
python mesh_filtering.py -- material/sfm_data.json material/scene_dense_mesh.obj output/ 50000
```

## How it works

### 1. Dominant axis estimation

The script reads the rotation matrices from the `extrinsics` section of `sfm_data.json` and builds a covariance-like matrix by summing outer products of the first row of each rotation matrix. Eigenvalue decomposition (via `numpy.linalg.eigh`) finds the **minimum-variance direction** the axis most orthogonal to all camera orientations, which is used as the normal of the filtering plane.

### 2. Camera point extraction

Camera positions are read directly from the `extrinsics` section of `sfm_data.json`, using the `center`field of each pose.

### 3. Projection onto the filtering plane

Each camera point is projected onto a plane perpendicular to the dominant axis (with a small `-1.0` offset copy) to build a 3D convex region that encloses the scene of interest.

### 4. Convex hull and planar section

PyMeshLab generates a **convex hull** from the projected camera points and then cuts it with a planar section aligned to the dominant axis, producing a 2D bounding polygon (`plane.obj`).

### 5. Mesh filtering

Every vertex of the dense mesh is projected onto the bounding plane and tested for containment inside the 2D polygon (using `matplotlib.path.Path.contains_points`). Only faces whose vertices all fall inside the polygon are kept.

### 6. Cleanup and decimation

The filtered mesh is cleaned by removing small disconnected components and split into connected components. The **largest component** is retained. If `num_polygons > 0`, quadric edge collapse decimation is applied to reduce face count while preserving shape.

### 7. Output

The final optimized mesh is saved to `<output_folder>/optimizated_mesh.obj`. The original `mesh_path` file is never modified.

---

## Output files

| File | Description |
|---|---|
| `<output_folder>/plane.obj` | Convex bounding polygon used for filtering |
| `<output_folder>/optimizated_mesh.obj` | **Final mesh** after cleanup and optional decimation |

The original `scene_dense_mesh.obj` is left untouched.

---

## Notes

- Decimation quality can be tuned by editing the `meshing_decimation_quadric_edge_collapse` parameters directly in the script (e.g. `qualitythr`, `preserveboundary`).
- The dominant camera axis is printed to stdout as a comma-separated string, which can be useful for debugging or downstream use.
- Views without an associated pose in `sfm_data.json` (i.e. unreconstructed images) are automatically skipped.

## References

**OpenMVG** - Open Multiple View Geometry  
A library for computer vision scientists and targeted at the Multiple View Geometry community. It provides a complete SfM pipeline, including feature extraction, matching, and camera pose estimation.  
→ [https://github.com/openMVG/openMVG](https://github.com/openMVG/openMVG)  
→ Pierre Moulon, Pascal Monasse, Renaud Marlet, and Others. *OpenMVG: Open Multiple View Geometry*. In Proceedings of the International Workshop on Reproducible Research in Pattern Recognition, 2016.

**OpenMVS** - Open Multi-View Stereo  
A library for recovering the full surface of a scene from a set of calibrated images. It takes the camera poses produced by OpenMVG and generates dense point clouds, meshes, and textured models.  
→ [https://github.com/cdcseacave/openMVS](https://github.com/cdcseacave/openMVS)
